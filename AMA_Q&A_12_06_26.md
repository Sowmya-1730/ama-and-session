# JavaScript & Python Interview Questions

## Why do we keep the `<script>` tag before the closing `</body>` tag?

Placing the `<script>` tag before the closing `</body>` tag ensures that the HTML content loads first before JavaScript executes.

This improves page loading speed and prevents JavaScript from trying to access HTML elements that have not yet been loaded.

Example:

```html
<body>

    <h1>Hello World</h1>

    <script src="app.js"></script>

</body>
```

## What is the use of `Array.flat()`?

`Array.flat()` is used to flatten nested arrays into a single array.

Example:

```javascript
const numbers = [
    1,
    2,
    [3, 4]
];

console.log(
    numbers.flat()
);
```

Output:

```javascript
[1, 2, 3, 4]
```

## What are Promise States?

A Promise has three states:

### Pending

The operation is still running.

### Fulfilled

The operation completed successfully.

### Rejected

The operation failed.

Example:

```javascript
const promise =
    new Promise(
        (resolve, reject) => {

            resolve(
                "Success"
            );

        }
    );
```

## How to Avoid Callback Hell?

Callback hell happens when many callbacks are nested inside each other.

Bad Example:

```javascript
task1(() => {

    task2(() => {

        task3(() => {

            console.log(
                "Done"
            );

        });

    });

});
```

To avoid callback hell:

* Use Promises
* Use Async/Await
* Split code into separate functions

## Difference Between Import and Require

### require()

Used in CommonJS modules (Node.js).

```javascript
const fs =
    require("fs");
```

### import

Used in ES Modules.

```javascript
import fs from "fs";
```

### Difference

| require             | import            |
| ------------------- | ----------------- |
| CommonJS            | ES Modules        |
| Synchronous         | Static Loading    |
| Node.js Traditional | Modern JavaScript |

## What is Asynchronous and Synchronous Execution?

### Synchronous

Tasks execute one after another.

```javascript
console.log("A");

console.log("B");

console.log("C");
```

Output:

```javascript
A
B
C
```

---

### Asynchronous

Tasks can execute later without blocking the program.

```javascript
console.log("A");

setTimeout(() => {

    console.log("B");

}, 1000);

console.log("C");
```

Output:

```javascript
A
C
B
```

## What Does `.map()` Return?

`.map()` returns a **new array**.

Example:

```javascript
const numbers =
    [1, 2, 3];

const doubled =
    numbers.map(
        number =>
            number * 2
    );

console.log(doubled);
```

Output:

```javascript
[2, 4, 6]
```

Return Type:

```javascript
Array
```

## Difference in Hoisting Between Function Declaration and Function Expression

### Function Declaration

```javascript
greet();

function greet() {

    console.log(
        "Hello"
    );

}
```

Output:

```javascript
Hello
```

Function declarations are fully hoisted.

---

### Function Expression

```javascript
greet();

const greet =
    function () {

        console.log(
            "Hello"
        );

    };
```

Output:

```javascript
ReferenceError
```

Function expressions are not fully hoisted.

## Event Handling is Possible in HTML. Why Use Event Listeners in JavaScript?

HTML Event:

```html
<button onclick="showMessage()">
    Click
</button>
```

JavaScript Event Listener:

```javascript
button.addEventListener(
    "click",
    showMessage
);
```

Advantages of Event Listeners:

* Keeps HTML clean
* Multiple listeners can be attached
* Easier maintenance
* Better separation of concerns

## What is Autoboxing?

Autoboxing is the automatic conversion of a primitive value into its corresponding object wrapper.

JavaScript Example:

```javascript
let name =
    "Sowmya";

console.log(
    name.length
);
```

Output:

```javascript
6
```

Even though `name` is a primitive string, JavaScript temporarily converts it into a `String` object so that methods and properties can be used.

This automatic conversion is called **Autoboxing**.
